### Data Imputation

#### 1. MissForest: non-parametric missing value imputation for mixed-type data

> [1] Stekhoven, D. J., & Bühlmann, P. (2012). MissForest—non-parametric missing value imputation for mixed-type data. *Bioinformatics*, *28*(1), 112-118. [pdf](material\ML\Data Imputation\btr597.pdf)  [web](https://academic.oup.com/bioinformatics/article/28/1/112/219101?login=true)

__kNN Imputation__:

kNN imputation: The idea is that observations (usually row in datasets) with closest ranges in feature space are most likely to share the same feature categories/values, and we can use the features from nearest neighbors to fill the missing values. In practice, train a kNN model on the observed values and fill the missing values using the trained model.

> Algorithm of Cross-Validation kNN Imputation:
>
> Prerequisite: a dataset __X__ with size n$\times$p, number of validation sets *l*, range of number of nearest neighbors __K__
>
> 1. $\bold{X}^{CV}$ &larr; initial imputation using mean imputation;
> 2. __for__ *t* in 1, ..., *l* __do__
> 3. ​    $\bold{X}_{mis, t}^{CV}$  &larr; artificially introduce missing values to $\bold{X}^{CV}$;
> 4. ​    for *k* in __K do__
> 5. ​        $\bold{X}_{kNN, t}^{CV}$  &larr; kNN imputation of $\bold{X}_{mis, t}^{CV}$ using *k* nearest neighbors;
> 6. ​        $\epsilon_{k, t}$ &larr; error of kNN imputation for *k* and *t*;
> 7. ​    __end for__
> 8. __end for__
> 9. $k_{best}$ &larr; $argmin_{k}\dfrac{1}{l}\sum_{t=1}^{l}\epsilon_{k, t}$;
> 10. $\bold{X}^{imp}$ &larr; kNN imputation of __X__ using $k_{best}$ nearest neighbors.



__MissForest__:	

> Algorithm of MissForest:
>
> Prerequisite: a dataset __X__ with size n$\times$p, stopping criterion $\gamma$
>
> 1. Fill the missing values using mean imputation or other simple imputation method;
> 2. __k__ &larr; vector of sorted indices of columns in __X__ with increasing number of missing values;
> 3. __while__ not $\gamma$ __do__
> 4. ​    $\bold{X}_{old}^{imp}$ &larr; store previously imputed dataset;
> 5. ​    __for__ *s* in __k do__
> 6. ​        Train a Random Forest using $\bold{y}_{obs}^{(s)}\sim\bold{x}_{obs}^{(s)}$;
> 7. ​        Predict $\bold{y}_{mis}^{(s)}$ using $\bold{x}_{mis}^{(s)}$;
> 8. ​        $\bold{X}_{new}^{imp}$ &larr; update the imputed dataset, where the missing values of feature *s* is replaced using $\bold{y}_{mis}^{(s)}$;
> 9. ​    __end for__
> 10. ​    update $\gamma$.
> 11. __end while__
> 12. __return__ the imputed dataset $\bold{X}^{imp}$

where during the imputation phase, dataset with index $\{1, 2, ..., n\}$ are divided into four parts:

1.  $\bold{y}_{obs}^{(s)}$ refers to the observed values in feature *s*, the corresponding index denote $\bold{I}_{obs}^{(s)}$;
2.  $\bold{y}_{mis}^{(s)}$ refers to the missing values in feature *s*, the corresponding index denote $\bold{I}_{mis}^{(s)}$; ($\bold{I}_{obs}^{(s)}\bigcup\bold{I}_{mis}^{(s)}=\{1, 2, ..., n\}$)
3.  $\bold{x}_{obs}^{(s)}$ refers to the subset of dataset with index $\bold{I}_{obs}^{(s)}$ with features all but *s*;
4.  $\bold{x}_{mis}^{(s)}$ refers to the subset of dataset with index $\bold{I}_{mis}^{(s)}$ with features all but *s*.



__The stopping criterion__: The stopping criterion $\gamma$ is calculated as the difference between the newly imputed dataset and stored previous dataset. The imputation phase stops when the difference for numerical/categorical features both increase.

The difference of numerical features __N__ is calculated as:
$$
\Delta_{N}=\dfrac{\sum_{j\in\bold{N}}(\bold{X}_{new, j}^{imp}-\bold{X}_{old, j}^{imp})^{2}}{\sum_{j\in\bold{N}}(\bold{X}_{new, j}^{imp})^{2}}
$$
And the difference of categorical features __C__ is calculated as:
$$
\Delta_{C}=\dfrac{\sum_{j\in\bold{C}}\sum_{i=1}^{n}\bold{I}_{\bold{X}_{new, i, j}^{imp}\neq\bold{X}_{old, i, j}^{imp}}}{\#_{NA}}
$$
where $\#_{NA}$ is number of missing values in categorical features __C__.



#### 2. Multiple imputation by chained equations: what is it and how does it work?

> [1] Azur, M. J., Stuart, E. A., Frangakis, C., & Leaf, P. J. (2011). Multiple imputation by chained equations: what is it and how does it work?. *International journal of methods in psychiatric research*, *20*(1), 40-49. [pdf](material\ML\Data Imputation\Int J Methods Psych Res - 2011 - Azur - Multiple imputation by chained equations  what is it and how does it work.pdf)  [web](https://onlinelibrary.wiley.com/doi/epdf/10.1002/mpr.329)

__Multiple Imputation by Chained Equations / MICE__:

> MICE algorithm:
>
> 1. Use some simple imputation method like mean imputation to fill in the missing values and store the missing matrix;
> 2. Set missing indices (imputed in step 1) of one feature (denote as "var") back to missing;
> 3. The observed values in feature "var" are regressed on other features in the imputation model;
> 4. The missing indices of feature "var" are replaced with predictions from regression model.
> 5. Repeat step 2-4 for all features with missing values. The cycling through the features for one time constitutes one iteration or "cycle". All missing values are replaced with predictions for one time in one cycle.
> 6. Repeat step 2-4 for a number of cycles, where the dataset is updated at each cycle.

The imputation model is under assumption where one can use Linear, Logistic or Poisson regression for imputation. Number of cycles needed for imputation is critical for accuracy of imputation, however, it determines runtime of imputation phase. So, it can be set as a hyperparameter and tuned by demands.

#### 4. The efficient imputation method for neighborhood-based collaborative filtering

> [1] Ren, Y., Li, G., Zhang, J., & Zhou, W. (2012, October). The efficient imputation method for neighborhood-based collaborative filtering. In *Proceedings of the 21st ACM international conference on Information and knowledge management* (pp. 684-693). [pdf](material\ML\Data Imputation\2396761.2396849.pdf)  [web](https://dl.acm.org/doi/pdf/10.1145/2396761.2396849)

The paper proposes a imputation procedure, <span style='color:red'>Auto-Adaptive Imputation (AutAI)</span>, which can significantly improve the accuracy of neighborhood-based collaborative filtering. The core idea of this method is that, not all values in the design matrix is equally valuable to predict/fill the missing values. So in the user$\times$items matrix (under recommendation system settings), those users provides information of same items should share similar preferences and their ratings for the target item (missing in the imputation problem) should be the main source.

To make a rate prediction in user$\times$items design matrix for user $u_{a}$ on item $t_{s}$, a imputed set (more significant for the specific user, item) is identified by related users $\mathcal{U}_{a}$ and related items $\mathcal{T}_{s}$:
$$
\mathcal{U}_{a}=\{u_{a'}|r_{a's}\neq\varnothing\}\\
\mathcal{T}_{s}=\{t_{i}|t_{i}\subset|T_{aa'_{1}}\cup...\cup T_{aa'_{i}}\cup ...T_{aa'_{l}}|\}
$$
where $r_{a's}$ is the rating of item $s$ by user $a'$ in the matrix, $T_{aa'_{i}}$ is the co-rated item set by user $a$ and $a'_{i}\in\mathcal{U}_{a}$.

Then, *key neighborhood* for user $u_{a}$ and item $t_{s}$ can be defined as $\mathcal{N}_{a,s}=\{r_{a'i}|u_{a'}\in\mathcal{U}_{a}, t_{i}\in\mathcal{T}_{s}\}$, where the value $r_{a'i}$ can be either observed or missing. All missing data in the *key neighborhood* is defined as *key set* of missing data for prediction $\hat{r}_{as}$. The region *key neighborhood* is crucial for prediction.

When *key neighborhood* $\mathcal{N}_{a,s}$ is determined for user $u_{a}$ and item $t_{s}$, all missing data should first be imputed as:
$$
\hat{r}_{a'i}=\bar{u}_{a'}+\dfrac{\sum_{u_{x}\in\mathcal{N}_{k}(u_{a'})}sim(u_{a'}, u_{x})\times(r_{xi}-\bar{u}_{x})}{\sum_{u_{x}\in\mathcal{N}_{k}(u_{a'})}sim(u_{a'}, u_{x})}
$$
where $sim(u_{a'}, u_{x})$ is the similarity between $u_{a'}$ and $u_{x}$, which can be expressed as:
$$
sim(u_{a'}, u_{x})=\dfrac{\sum_{t_{i}\in T_{a'x}}(r_{a'i}-\bar{u}_{a'})(r_{xi}-\bar{u}_{x})}{\sqrt{\sum_{t_{i}\in T_{a'x}}(r_{a'i}-\bar{u}_{a'})^{2}\sum_{t_{i}\in T_{a'x}}(r_{xi}-\bar{u}_{x})^{2}}}
$$
which is the *Pearson Correlation Coeffcient* between $u_{a'}$ and $u_{x}$.

When $\mathcal{N}_{a, s}$ imputed, these information can be used to predict for  user $u_{a}$ and item $t_{s}$.

>Auto-Adaptive Imputation (AutAI) algorithm:
>
>For a user-item matrix $\mathcal{R}$, to predict for  user $u_{a}$ and item $t_{s}$:
>
>1. __for__ each $u_{x}\in\mathcal{U}$ __do__
>2.   __for__ each $u_{y}\in\mathcal{U}$ & $u_{y}\neq u_{x}$ __do__
>3. ​    calculate the $sim(u_{x},u_{y})$; 
>4.   __end for__
>5. __end for__
>6. $\mathcal{R'}\leftarrow\mathcal{R}$;
>7. $\mathcal{U}_{a}\leftarrow\{u_{a'}|r_{a's}\neq\varnothing\}$;
>8. $\mathcal{T}_{s}\leftarrow\varnothing$;
>9. __for__ each $u_{a'}\in\mathcal{U}_{a}$ __do__
>10.   $\mathcal{T}_{aa'}\leftarrow\{t_{i}|r_{ai}\neq\varnothing, r_{a'i}\neq\varnothing\}$;
>11.   $\mathcal{T}_{s}\leftarrow\mathcal{T}_{s}\cup\mathcal{T}_{aa'}$;
>12. __end for__
>13. $\mathcal{N}_{a, s}\leftarrow\{r_{a'i}|u_{a'}\in\mathcal{U}_{a}, t_{i}\in\mathcal{T}_{s}\}$;
>14. __for__ each $r_{a'i}\in\mathcal{N}_{a, s}$ __do__
>15.   __if__ $r_{a'i}\neq\varnothing$ __then__
>16. ​    $\hat{r}_{a'i}\leftarrow\bar{u}_{a'}+\dfrac{\sum_{u_{x}\in\mathcal{N}_{k}(u_{a'})}sim(u_{a'}, u_{x})\times(r_{xi}-\bar{u}_{x})}{\sum_{u_{x}\in\mathcal{N}_{k}(u_{a'})}sim(u_{a'}, u_{x})}$;
>17. ​    $\mathcal{R}'(u_{a},t_{i})\leftarrow\hat{r}_{a'i}$;
>18.   __end if__
>19. __end for__
>20. __return__ $\mathcal{R}'$;

#### 8. Missing value imputation: a review and analysis of the literature (2006–2017)

> [1] Lin, W. C., & Tsai, C. F. (2020). Missing value imputation: a review and analysis of the literature (2006–2017). *Artificial Intelligence Review*, *53*(2), 1487-1509. Two major strategies can be applied to evaluate the imputation process: (1) directly compare the imputed datasets and orginal datasets (given the orignal datasets are non-missing and available); (2) train a model on the imputed datasets and compare the classification accuracy/regression errors.



### Data Balancing

A study of the behavior of several methods for balancing machine learning training data. *ACM SIGKDD explorations newsletter*, *6*(1), 20-29. Springer, Berlin, Heidelberg. [pdf](material\ML\Data Balancing\2001_Book_ArtificialIntelligenceInMedici.pdf)  [web](https://link.springer.com/chapter/10.1007/3-540-48229-6_9)

>
> [5] Chawla, N. V., Bowyer, K. W., Hall, L. O., & Kegelmeyer, W. P. (2002). SMOTE: synthetic minority over-sampling technique. *Journal of artificial intelligence research*, *16*, 321-357. [pdf](material\ML\Data Balancing\10302-Article Text-18973-1-10-20180216.pdf)  [web](https://www.jair.org/index.php/jair/article/view/10302)

Sometimes, the dataset is unbalanced, where number of one class (denotes majority class) heavily outnumber all other class (denotes minority class) combined. The case is especially common in insurance datasets. For example, for a claim report dataset, majority of policyholders report no claim. The unbalanced nature can affect the model's accuracy, since the weights of minor class is insignificant compared to majority class. For the above example, a simple model which predicts no claim for all policyholders still can reach high overall prediction accuracy, while the False Negative (refers to has a claim but reported as no claim) ratio can be high. For insurers, the goal is always to accurately predict the claim numbers and claim sizes, which are usually the minor class observations. And in practice, sometimes, the minor class can be more critical, as they potentially represent the higher risk or major financial impact. In this case, a data balancing method is critical to remove (at least reduce) the negative impact brought by unbalance nature and increase the model's true accuracy.

Two of the most frequently-used balancing methods are over-sampling and under-sampling, where the goal is to increase the minority percentage by synthesizing minority class samples and to reduce majority percentage by removing majority class samples. All below methods can be categories as variations of over-sampling or under-sampling, or the combination of over-sampling and under-sampling.



__Simple Random Over Sampling__:

The Simple Random Over Sampling tries to replicate random samples selected from minority samples into the dataset. Or the method can be view as increase the weights of minority class samples instead of equally weighting all samples.

__Simple Random Under Sampling__:

The Simple Random Under Sampling tries to randomly eliminate samples from majority class, so that the majority class percentage is reduced.

Both Simple Random Over Sampling and Simple Random Under Sampling have not learnt new knowledge from the dataset, and sometimes, the replicated minority samples may not be decision boundary significant or even holds negative impact on learning decision boundary in the over-sampling method (or, the eliminated majority samples may be critically important for decision boundary learning in the under-sampling method).

__Tomek Link__:

<p align="center">
    <img src="material\ML\Data Balancing\A_study_of_the_behavior_of_several_methods_for_balancing_machine_learning_training_data_1.png" alt="Tomek_link" style="zoom:65%"/>
</p>

<div style="text-align: center"> Fig. Tomek Link </div>

Tomek Link is an under-sampling method. Tomek Link refers to two samples $E_{i}$ and $E_{j}$ that belong to two different classes (majority and minority in the case) and distance represented as $d(E_{i}, E_{j})$, where there exists not $E_{l}$ among all observations that satisfies $d(E_{i}, E_{l})<d(E_{i}, E_{j})$ or $d(E_{j}, E_{l})<d(E_{i}, E_{j})$. Or in words, Tomek Link are the two nearest neighbors that belong to different classes. Once Tomek Links are determined, those majority samples in the pairs are eliminated. As shown in above figure, the orange squares represent minority samples and blue circles represent majority samples in the left part and Tomek Links are determined in the middle, then the majority samples are removed. 

Tomek Link is designed to remove noise observations and decision boundary significant observations. As shown in the above figure, it's clear that after Tomek Link, the decision boundary for the majority/minority class is much clearer than before.

__Edited Nearest Neighbor__:

<p align="center">
    <img src="material\ML\Data Balancing\A_study_of_the_behavior_of_several_methods_for_balancing_machine_learning_training_data_2.png" alt="Edited_Nearest_Neighbor" style="zoom:50%"/>
</p>

<div style="text-align: center"> Fig. Edited Nearest Neighbor </div>

Edited Nearest Neighbor (ENN) [2] is an under-sampling method. ENN removes any examples whose class differs from the class of at least two of its three nearest neighbors.

__Neighborhood Cleaning Rule__:

Neighborhood Cleaning Rule (NCL) [3] is a modification of ENN to increase the data cleaning efficiency since ENN may remove minority class which result in longer runtime. Instead of removing all samples disagreed by nearest neighbors, NCL defines that if the randomly selected sample $E_{i}$ belongs majority class and the classification given by 3 nearest neighbors contradicts with the true classification, $E_{i}$ will be removed; if the randomly selected sample $E_{i}$ belongs minority class and classification predicted by 3 nearest neighbors contradicts with the true classification, nearest neighbors that belong to majority class will be removed. NCL tends to retain all minority samples and only remove majority class (believe all minority samples are correctly labeled), thus provides a faster balancing procedure but may keep potentially inaccurately labeled minority samples (some minority samples may in the majority part of feature space but NCL tends to keep these samples).

__Condensed Nearest Neighbor__:

Condensed Nearest Neighbor (CNN) [4] is a under-sampling method which is used to find the consistent subset of samples. The subset $\hat{E}\subseteq E$  is consistent with $E$ that if using 1-NN, $\hat{E}$ can correctly classifies all samples in $E$. The subset $\hat{E}$ is constructed as: First, randomly select one majority sample and all samples from minority class as $\hat{E}$. Then, predict all samples in $E$ using $\hat{E}$ with 1-NN model. And put all misclassified samples in $E$ into $\hat{E}$. 

The idea of CNN is to remove those samples in majority class that are distant from decision boundary (only distant samples will be correctly predicted by the first constructed $\hat{E}$), since they are less relevant in finding decision boundary.

__One Sided Selection__:

One Sided Selection (OSS) is a combined under-sampling method, which applied Tomek Link followed by CNN (or can be written as Tomek Link-CNN). OSS can take advantages of both methods, where the first step (Tomek Link) removes noise and decision boundary significant samples, and second step (CNN) removes observations distant from decision boundary. In a two-step cleaning method, the balanced dataset is better shaped.

__CNN-Tomek Link__:

CNN-Tomek Link is similar to OSS but with a change of order of two steps. Since for Tomek Link, computing distances between all samples is required (to get nearest neighbors), which is computational expensive, so, applying CNN to reduce number of observations first can reduce the calculation needed for Tomek Link. To that sense, CNN-Tomek Link can clean the dataset faster compared to OSS while still take advantages of both methods.

__Smote__:

<p align="center">
    <img src="material\ML\Data Balancing\A_study_of_the_behavior_of_several_methods_for_balancing_machine_learning_training_data_3.png" alt="Smote" style="zoom:60%"/>
</p>

<div style="text-align: center"> Fig. Smote </div>

Synthetic Minority Over-Sampling Technique (Smote) [5] is a over-sampling method where the idea is to generate synthetic minority samples. The synthetic minority samples are spatially interpolated among existed minority samples. The process of Smote is demonstrated in above figure. One problem is that sometimes, by interpolation among minority samples, the decision boundary of minority class may spread into the majority class in feature space, which can lead to overfitting.

__Smote-Tomek Link__:

<p align="center">
    <img src="material\ML\Data Balancing\A_study_of_the_behavior_of_several_methods_for_balancing_machine_learning_training_data_4.png" alt="Process_of_Smote_Tomek_Link" style="zoom:60%"/>
</p>

<div style="text-align: center"> Fig. (a) original dataset; (b) over-sampled dataset; (c) Tomek Link identification; (d) Under-sampling and balanced dataset. </div>

Smote-Tomek Link is a combined over- and under-sampling method. One problem with Smote is that synthetic minority samples generated can expand deeply into the majority class realm. Thus, applying Tomek Link after over-sampling using Smote can remove those noise and decision boundary significant samples. The process of Smote-Tomek Link is shown in above figure.

__Smote-ENN__:

The motivation of Smote-ENN is the same as Smote-Tomek Link. However, since ENN provides deeper cleaning, Smote-ENN can yield more in depth cleaning.



### Feature Selection

#### 1. Adaptive floating search methods in feature selection

> [1] Somol, P., Pudil, P., Novovičová, J., & Paclık, P. (1999). Adaptive floating search methods in feature selection. *Pattern recognition letters*, *20*(11-13), 1157-1163. [pdf](material\ML\Feature Selection\1-s2.0-S0167865599000835-main.pdf) [web](https://www.sciencedirect.com/science/article/pii/S0167865599000835)

__Sequential Forward Selection / SFS__: Starting with empty selection rule, adding one best-performing feature at a time until reaches best-performing feature subset.

__Sequential Backward Selection / SBS__: Starting with full feature list, removing one most irrelevant feature at a time until best-performing feature subset selected.



__Classical Floating Search / CFS__: The idea of floating introduced is that at forward step, the algorithm adds new features to the subset (forward search, remove features for backward search) and at backward step, the algorithm removes some of the new features if performs better (forward search, add features for backward search). The "classical" refers to pre-defined number of features per step. However, appropriate choice of the number is difficult to determine, which makes the method not feasible.

__Sequential Forward Floating Search / SFFS__ and __Sequential Backward Floating Search / SBFS__

<p align="center">
    <img src="material\ML\Feature Selection\Adaptive_floating_search_methods_in_feature_selection_1.png" alt="SFFS" style="zoom:60%"/>
</p>

<div style="text-align: center"> Fig. SFFS </div>



__Adaptive Floating Search / AFS__:  To overcome the problem with CFS, adaptive step *r* is introduced, which is defined as:

<p align="center">
    <img src="material\ML\Feature Selection\Adaptive_floating_search_methods_in_feature_selection_2.png" alt="Adaptive_step" style="zoom:50%"/>
</p>

<div style="text-align: center"> Fig. Adaptive step </div>

1. if $\abs{k-d}<b$, let $r=r_{max}$
2. elif $\abs{k-d}<b+r_{max}$, let $r=r_{max}+b-\abs{k-d}$
3. else let $r=1$

where *d* is the desired number of features selection, *b* denotes neighborhood of the selection, *k* is the number of features currently selected, $r_{max}$ is the maximum number of features allowed to change per step.



__Adaptive Sequential Forward Floating Search / ASFFS__: 

<p align="center">
    <img src="material\ML\Feature Selection\Adaptive_floating_search_methods_in_feature_selection_3.png" alt="ASFFS" style="zoom:50%"/>
</p>

<div style="text-align: center"> Fig. ASFFS </div>

The algorithm is initialized at current feature subset $k=0$, and empty selection rule $X_{0}=\empty$, desired number of features selected $D$. Denotes so-far best values of criterion function $J(X_{i})$ are stored as $J_{i}^{max}$ for $i=1, 2, ..., D$.

__Forward Phase__: 

At the beginning of the forward phase, adaptive step *r* is recalculated using above function.

> 1. Let $o=1$.
> 2. (Conditional inclusion) select most significant *o*-tuple $U_{o}^{m}$ from the set of available features $X\setminus X_{k}$, and add the tuple of features to $X_{k}$ to form new feature subset $X_{k+o}$.
> 3. If $J(X_{k+o})>J_{k+o}^{max}$ (the newly formed feature subset outperforms the stored ones), let $k=k+o$ and go to step 6.
> 4. (Conditional increase of generalization step) If $o<r$, let $o=o+1$ and go to step 2. (The newly formed feature subset performs worse than the stored ones, increase the searching range.)
> 5. If can't find any feature subset performs better than the stored ones in searching range of *r*, forget the current feature subset $X_{k}$. Let $k=k+1$, and use the stored best feature subset for new *k* as current feature subset $X_{k}$.
> 6. (Termination condition) If $k\geq d+\Delta$, stop the algorithm.

__Backward Phase__:

At the beginning of the forward phase, adaptive step *r* is recalculated using above function.

> 7. Let $o=1$.
>
> 8. (Conditional exclusion) select the least significant *o*-tuple $T_{o}^{l}$ in current feature subset $X_{k}$, and remove the tuple of features from $X_{k}$ to form new feature subset $X_{k-o}$.
>
> 9. If $J(X_{k-o})>J_{k-o}^{max}$ (the newly formed feature subset outperforms the stored ones), let $k=k-o$ and restart the __Backward Phase__ (to check whether removing features can result in better selection).
>
> 10. (Conditional increase of generalization step) If $o<r$, let $o=o+1$ and go to step 2.
>
> 11. If can't find any feature subset performs better than the stored ones in searching range of *r*, go to __Forward Phase__.



#### 2. A genetic algorithm-based method for feature subset selection

> [1] Tan, F., Fu, X., Zhang, Y., & Bourgeois, A. G. (2008). A genetic algorithm-based method for feature subset selection. *Soft Computing*, *12*(2), 111-120. [pdf](material\ML\Feature Selection\Tan2008_Article_AGeneticAlgorithm-basedMethodF.pdf)  [web](https://link.springer.com/article/10.1007/s00500-007-0193-8)

Genetic Algorithm (GA) is an algorithm inspired by the process of nature selection and belongs to a larger class Evolutionary Algorithm (EA). Common GA consists of three steps: Selection, Crossover, Mutation. The paper introduces a feature selection method using GA.



__Algorithm of GA__:

<p align="center">
    <img src="material\ML\Feature Selection\A_genetic_algorithm-based_method_for_feature_subset_selection_1.png" alt="Procedure of GA" style="zoom:45%"/>
</p>

<div style="text-align: center"> Fig. Procedure of GA </div>

1. *Feature Pool*: feature pool is a set of feature selection rules which stores the initial rules and those generated by the algorithm.

   Three initial feature selection rule generation methods:

   1. *Entropy-based feature ranking*: The algorithm is based on the fact that orderly configurations have lower entropy, which in a feature selection scenario, removing irrelevant features will reduces entropy more than removing relevant features. The features can be ranked the entropy of removing one feature at a time, and the highest ranks indicate more relevant features. The feature selection rule can be generated by removing one feature each time, calculating corresponding entropy and reserving highest ranks features. The entropy of a dataset with *N* observations can be expressed as:
      $$
      E=-\sum_{i=1}^{N}\sum_{j=1}^{N}s_{ij}\log{s_{ij}}+(1-s_{ij})\log{(1-s_{ij})}
      $$
      where similarity $s_{ij}=e^{-\alpha D_{ij}}$ between two observations $x_{i}$ and $x_{j}$ are calculated using Euclidean distance $D_{ij}$ and parameter $\alpha$ calculated as $\alpha=\dfrac{-\ln0.5}{\bar{D}}$ where $\bar{D}$ is the average distance among all observations. The method is of unsupervised form since no response information required.

   2. *t-statistics*: The t-statistics are calculated for each feature and highest ranks of features with highest t-statistics are selected for the feature selection rule. For a two-class dataset, denote each feature *s*, mean of the two classes as $\mu_{s}^{1}$ and $\mu_{s}^{2}$ and standard deviation of the two classes $\delta_{s}^{1}$ and $\delta_{s}^{2}$, the t-statistics score of *s* is:
      $$
      T(s)=\dfrac{\abs{\mu_{s}^{1}-\mu_{s}^{2}}}{\sqrt{\dfrac{(\delta_{s}^{1})^{2}}{n_{1}}+\dfrac{(\delta_{s}^{2})^{2}}{n_{2}}}}
      $$
      where $n_{1}$ and $n_{2}$ are the number of observations belonging to the two classes respectively.

   3. *SVM-RFE*: Support Vector Machine-Recursive Feature Elimination is a method to train a SVM model on the dataset, compute the ranking criterion of all features, and recursively remove the feature with the smallest ranking criterion. Since the model is trained using SVM and includes recursive elimination, the efficiency of the method is not guaranteed.

   After the initialization, there will be a collection of feature selection rules in the feature pool to start with. To ease the memory, the selection rule can be stored as a {0, 1} list where 1 indicates the feature being selected and 0 being discarded.

2. *Fitness function*: Since number of features being selected by the rule is critical in the algorithm (or usually more feature gets better model accuracy), the algorithm is designed to optimize two objects: maximize the model accuracy and minimize the number of features selected, which is measured by a defined fitness function:
   $$
   F(x)=w*c(x)+(1-w)*\dfrac{1}{s(x)}
   $$
   where *x* is a feature selection rule, $w$ is a weight in [0, 1], $c(x)$ is the model accuracy and $s(x)$ is size of the feature selection rule (like number of features selected). For fitness function, increasing model's accuracy and reducing number of features can maximize the function value.

3. *Induction algorithm*: The induction algorithm is used to measure the model accuracy, and is independent of GA algorithm. For this paper, the SVM classifier is used. (The model accuracy can incorporate into the fitness function $c(x)$ part.)

4. *Genetic operators*: This is the actual part of GA algorithm, which generates generations of children feature selection rules, evaluated and selected.

   1. *Selection*: Roulette wheel selection is used to probabilistically select rules from the feature pool. The probability of individual rules being selected is calculated as weighted values of fitness function. For individual rule $h_{i}$, the probability of the rule being selected is
      $$
      P(h_{i})=F(h_{i})/\sum_{i=1}^{P}F(h_{i})
      $$
      where $F(h_{i})$ denotes fitness function of $h_{i}$. With the probability assigned to each selection rule, few parents rules are picked from the pool.

   2. *Crossover*: 

      <p align="center">
          <img src="material\ML\Feature Selection\A_genetic_algorithm-based_method_for_feature_subset_selection_2.png" alt="Single_point_crossover" style="zoom:45%"/>
      </p>

      <div style="text-align: center"> Fig. Single-point Crossover </div>

      Crossover operators are inspired by recombination process where two parent DNA combine genetic information to generate new offspring (children DNA). In GA, parts of selection rules are exchanged during crossover, with the hope that two well-performing selection rule sections are combined after the crossover and get better-performing children rules.
   
   3. *Mutation*: Like mutation in nature biology, mutation occurs in GA. During this phase, *n* bits have the probability of $p_{m}$ of flipping. Since the selection rule is a [0, 1] rule, flipping a bit means turn 0 into 1 (not select the feature into select the feature) or 1 into 0 (select the feature into not select the feature). Similar to natural mutation, the mutation probability is small, and the mutation does not guarantee generating better-performing selection rules but only possibility of a better rule.

With generations of GA and multiple selection rules will be stored in the feature pool, performance stability will be reached, and we can select the best-performing rule (according to fitness function) as final feature selection suggestion.

__Architecture Optimization__: Architecture Optimization (AO) is the step to find best-performing network structure hyperparameters based on the search space previously defined.

1. *Evolutionary algorithm*:  Evolutionary Algorithm (EA) is a generic population-based optimization algorithm inspired by biology evolution, and a mature global optimization method with high robustness and broad applicability.

   A typical EA comprises four steps: selection, crossover, mutation, and update:

   1. Selection: The step involves selecting portion of all generated networks as parent networks for later crossover step. Three common selection strategies: (1) *fitness selection*: the probability of a network being selected is proportional to its fitness value, $P(h_{i})=\dfrac{Fitness(h_{i})}{\sum_{j}Fitness(h_{j})}$, where $h_{i}$ indicates the *i*th network; (2) *rank selection*: rank the networks according to relative fitness values and assign probabilities accordingly; (3) *tournament selection*: at each iteration, *k* networks are randomly selected from the population and sorted according to performance, best network is assigned with probability *p*, second best *(1-p)$\times$p*, and so on.
   2. Crossover: During this stage, each two parent networks are selected to generate a new offspring network, which is inspired by genetic recombination. The genetic information of parent networks are encoded and passed to offspring networks.
   3. Mutation: Similar to gene mutation, this step involves randomly, independently flipping genetic information like whether enables connections between two layers, altering learning rate, to enrich the network pool and allow to explore novel structures. Though, mutation do not guarantee a better-performing network, it allows the possibility of generating never-seen network structure and may get better performance.
   4. Update: With each iteration of EA, new network structures are generated and put into a pool for recording and ready for next iteration. However, given limitation of memory, computational resources, removing oldest or worst-performing network structures are commonly used in the step.
